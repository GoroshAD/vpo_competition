Анализ событий Windows
Инциденты информационной безопасности с периодичностью могут возникать на рабочих станциях или серверах Windows, которые не подключены к SIEM системе на момент возникновения подозрения об инциденте по разным причинам, к тому же, в добавок может не быть конкретного контекста, что вызвало подозрение, в таких случаях может потребоваться анализ большого количества журналов событий. Для более оптимального подхода к анализу выберем события, которые в первую очередь стоит изучать.

Первоочередный анализ событий из журнала Security (безопасность).

Будем перечислять события по идентификаторам (EventID):

4688 — событие создания процесса. Данное событие полезно для выявления создания нелегитимных процессов, для отслеживания запуска хакерских утилит, выявления нетипичных командных строк, выявления аномалий между родительским и дочерним процессом. По умолчанию аудит данного события в системе не настроен.
4624 — событие успешной аутентификации. Событие полезно для анализа нелегитимных входов пользователей, которые не должны аутентифицироваться на конкретном хосте, либо для выявления откуда исходило заражение.
4625 — событие неуспешной попытки аутентификации. Событие полезно для выявления попыток подбора пароля.
1102 — событие об очистке журнала событий Security. 
4697 — событие установки службы. Событие полезно для выявления закрепления злоумышленника в системе через службы Windows.
4648 — событие попытки входа под заданной учетной записью. Активность может быть сгенерирована, когда процесс выполняется в режиме “Запуск от имени” (run as). В нормальном режиме работы систем данное событие встречается достаточно редко.

DLL-библиотеки могут быть:
Системными — содержат код, необходимый большинству системных приложений. Например:
Kernel32.dll — предоставляет базовые функции ядра: управление памятью, операции ввода-вывода, создание процессов, потоков и функции синхронизации.
User32.dll — реализует компонент Windows USER, который создаёт и управляет стандартными элементами пользовательского интерфейса Windows, такими как рабочий стол, окна и меню.
Предоставляемыми производителем стороннего программного обеспечения — например, libvlc.dll — часть VLC Media Player, устанавливается в системе вместе с .exe VLC.

Автоматическое применение изменений — при обновлении библиотеки DLL изменения автоматически применяются к каждому приложению, использующему этот файл.
